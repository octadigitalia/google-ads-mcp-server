## 1. Overview
This specification defines how the Google Ads MCP Server interacts with the LLM (Client) and the End User, focusing on standardized outputs and the integration of the **Native Field Linter (Story 9.4)**.

### 1.1 Goals
- Standardize tool outputs for better LLM parsing.
- Implement proactive error messaging via the GAQL Linter.
- Ensure consistent handling of Google Ads IDs and Enums.

## 2. Requirements Summary

### 2.1 Functional Requirements (Interface)
| ID | Description | Priority |
|----|-------------|----------|
| FR-1 | All tool results must include a `metadata` envelope with `customer_id`. | P0 |
| FR-2 | GAQL errors must return a "Suggested Correction" based on the Linter. | P0 |
| FR-3 | Enums must always be returned as Strings, not Integers. | P0 |
| FR-4 | IDs must be returned as Strings to prevent precision loss. | P0 |

### 2.2 Non-Functional Requirements
| ID | Category | Requirement | Metric |
|----|----------|-------------|--------|
| NFR-1 | Usability | Linter suggestions must be actionable. | 1-click fixable by LLM. |
| NFR-2 | Performance | Linter check must add < 200ms to query time. | latency_ms < 200. |

## 3. Interaction Design

### 3.1 Standard Success Output
Every query tool (e.g., `search_ads`, `get_campaign_performance`) must follow this JSON structure:

```json
{
  "metadata": {
    "customer_id": "1234567890",
    "row_count": 5,
    "query": "SELECT ...",
    "warnings": []
  },
  "results": [
    { "campaign.id": "123", "campaign.name": "Test" }
  ]
}
```

### 3.2 Linter Error Output (Story 9.4)
When an invalid field or resource combination is detected, the server must return:

```json
{
  "status": "ERROR",
  "error_code": "INVALID_GAQL_FIELD",
  "message": "Field 'campaign.invalid_field' does not exist in resource 'campaign'.",
  "suggestions": {
    "closest_match": "campaign.status",
    "valid_fields_sample": ["campaign.id", "campaign.name", "campaign.status"]
  }
}
```

## 4. Technical Approach

### 4.1 Linter Integration
The Linter will act as a middleware in `search_ads`. 
- **Step 1**: Parse the `SELECT` and `WHERE` clauses using regex.
- **Step 2**: Cross-reference fields against `docs/architecture/api-audit-*.json` (cached).
- **Step 3**: If fields are invalid, abort API call and return the Error Output.

### 4.2 Enum & ID Serialization
Using `src/mcp_server/utils.py`:
- `preserving_proto_field_name=True`
- `use_integers_for_enums=False`
