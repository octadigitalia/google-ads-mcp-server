## Story
**As a** Developer,
**I want** to implement the helper `resolve_resource_id` and apply it to write tools,
**so that** I can save tokens (LLM doesn't need to list resources to find IDs) and reduce "id hallucination" errors.

## Acceptance Criteria
1. [x] Create function `resolve_resource_id(customer_id, type, name_or_id)` in `utils.py`.
2. [x] Support types: `CAMPAIGN`, `AD_GROUP`, `USER_LIST`.
3. [x] If `name_or_id` is numeric or looks like a resource name, return as is.
4. [x] If it's a descriptive string, search via GAQL by exact or partial name.
5. [x] If multiple results are found, return a standardized ambiguity error with the list of found IDs.
6. [x] Update `set_campaign_status` and `set_campaign_budget` to use this helper.
